# E003: First hyperparameter sweep (6 profiles)

**Date:** 2026-03-15/16
**Duration:** ~15 min each (two runs, pre- and post-bitwise)
**Target:** R(5,5) n=25, leaderboard capacity 2000
**Logs:** `logs/fleet-20260315-234731/` and `logs/fleet-20260316-001819/`

## Config
16 workers distributed across 6 profiles:
- narrow-deep: beam=50, depth=20, bias=0.5
- standard: beam=100, depth=10, bias=0.5
- wide-shallow: beam=200, depth=5, bias=0.5
- ultra-wide: beam=400, depth=3, bias=0.5
- focused: beam=80, depth=12, bias=0.8
- explorer: beam=120, depth=8, bias=0.2

## Results (averaged across both runs)

| Profile | Admit/Wk | Disc/Rnd |
|---------|----------|----------|
| **focused** | **~245** | ~9,600 |
| standard | ~128 | ~5,680 |
| explorer | ~88 | ~3,640 |
| narrow-deep | ~60 | ~18,700 |
| wide-shallow | ~37 | ~1,200 |
| ultra-wide | ~0 | ~280 |

## Analysis
- **focused (bias=0.8) dominates** — 2-3x better than standard
- Top-heavy leaderboard sampling is the key factor
- ultra-wide is dead — beam=400 depth=3 produces nothing useful
- narrow-deep has highest disc/round (18K!) but low admissions (mostly duplicates)

## Decision
- Kill ultra-wide profile
- Add focused variants (focused-deep, focused-wide) to next sweep
- bias=0.8 (top-heavy sampling) is the most important hyperparameter
